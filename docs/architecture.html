Frontend

Next.js 16.1 App Router with NextAuth v5 (JWT sessions). Typed API client (PlatformApi) with SWR hooks for data fetching plus an observability layer that routes product analytics through PostHog and browser error, trace, and replay monitoring through Sentry.

Backend

Rust / Axum HTTP server. SaaS routes handle auth, runs, connections, dashboard, settings, audit, integrations, and webhooks. Event routes handle ingest, stream, and query, with an optional Helix-backed graph read model for vector and traversal workloads.

Type Safety

Rust structs generate TypeScript types via ts-rs. Generated files live in packages/shared/src/generated/. Run scripts/generate-types.sh to regenerate.

Infrastructure

BACKEND_MODE=memory uses DashMap (dev). BACKEND_MODE=real uses PostgreSQL via SQLx (production). EVENTS_BACKEND_MODE=helix keeps Postgres canonical in phase 1 while mirroring event graphs, typed payload nodes, traces, and embeddings into HelixDB. Kafka streaming is feature-gated and optional.

External Integrations

Nango provides OAuth and sync orchestration for supported integrations. Stripe handles payments and subscriptions. PostHog receives frontend analytics events through the app's vendor-agnostic analytics provider, and Sentry receives frontend browser errors, traces, and replay data.

Deployment

Backend deploys to Fly.io and the frontend apps deploy through app-level Vercel configs. Local Docker workflows live in deploy/docker-compose.yml and backend/deploy/docker-compose.yml.
