# Chronicle Design System

> A unified design language for Chronicle — the infrastructure layer
> for agent behavior.

A single Linear-aligned chrome — pitch-black canvas, graphite cards,
porcelain ink, ember as the one-hot signal — built on Radix UI
primitives, Tailwind CSS, and `cva`. shadcn-shaped where it matters
so paste-in upstream snippets resolve cleanly without losing the
Chronicle voice.

## Three principles

1. **Serif for judgement. Mono for evidence.**
   Display copy lands in Kalice; metrics, IDs, timestamps, and
   commands land in Berkeley Mono. Body text is Inter Variable.
2. **One hot surface. Everything else rests.**
   Ember (`--c-ember`, `#d8430a`) is the Chronicle signal. It marks
   the selected row, the primary CTA, the active version — and
   nothing else. If the whole UI glows, nothing glows.
3. **Streams are colored. Systems are quiet.**
   Event streams each own a hue (teal for support, amber for
   commerce, green for billing/ok). System chrome stays neutral so
   your eye tracks causality.

## Navigating this book

- **Foundations** — tokens: color, typography, spacing, motion,
  gradient/glass recipes, density anatomy.
- **Brand** — logo lockups and brand surfaces.
- **Typography** — `Display` / `Body` / `Mono` text primitives.
- **Surfaces** — the brand light-source + glass-pane treatments.
- **Primitives** — Button, Input, Card, Dialog, AlertDialog, Sheet,
  Command, Sonner, and the rest of the shadcn-shaped atoms.
- **Icons** — Lucide glyph index + Chronicle brand icons.
- **Layout** — `AppShell`, `TopBar`, `Sidebar`, `PageHeader`.
- **Auth / Onboarding / Connections / Connectors** — flow screens.
- **Datasets / Agents / Backtests** — domain composites.
- **Stream Timeline / Env Manager** — operator surfaces.
- **Product** — composed dashboards and event streams.

Every story has a **theme** switch in the toolbar — dark is the brand
default; light is the Chronicle-tuned cool-white alt.
