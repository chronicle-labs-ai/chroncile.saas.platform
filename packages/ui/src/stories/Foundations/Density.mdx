# Density — Brand vs Product

The Chronicle design system ships **two density layers** that share a
single token vocabulary but flip on intent.

- **Brand** — `--c-*` namespace. Marketing, decks, brand pages,
  onboarding splash, hero.
- **Product** — `--l-*` namespace. The app: dashboards, lists,
  editors, the timeline.

Both layers respect `data-theme="dark" | "light"`. Switching themes
flips both palettes in the same cascade pass — no remount, no flash.

---

## Brand density (`--c-*`)

Wider rhythm, generous padding, mono‑uppercase action labels. Reads
like an editorial spread. Every primitive in `packages/ui` defaults
to this when you reach for `density="brand"`.

- Button: 28 / 36 / 44 px tall, mono‑uppercase, `tracking-tactical`.
- AppShell: bordered card, 240 px nav, 52 px topbar, 64 px footer.
- Sidebar: `Eyebrow` mono‑uppercase section labels.
- Background: `bg-page` (the warm dark canvas).

Use it for: marketing site, principles, the Foundations section of
this book itself, and any surface where _calmness wins over speed_.

## Product density (`--l-*`)

Linear‑inspired. Sans‑medium 12.5 – 13 px copy, 28 / 32 / 36 px tall
controls, hairline 1 px borders, ember as the **single hot signal**.

- Button: 28 / 32 / 36 px tall, sans‑medium, `tracking-normal`,
  inline `<Kbd>` shortcut hints.
- AppShell: full‑bleed `bg-l-surface`, 44 px topbar, 224 px nav, no
  outer card. Optional `filterBar` slot between topbar and content.
- Sidebar: workspace switcher pinned top → `Workspace` → `Views` →
  `Saved Filters` → `Sources`. Active row gets an ember left bar.
- New chrome: `Chip`, `FilterPill` (`dim | verb | val | ×`), `Kbd`,
  `Priority` glyph, `Status` circle, `Label` pill, ⌘K palette,
  draggable minimap, sticky group head, inspector overlay drawer.

Use it for: every product surface — dashboard, sandbox, the event
timeline, replay suite, env‑manager.

---

## Reaching for tokens

In Tailwind you have parallel utilities for each layer:

```tsx
// Brand surface
<div className="bg-surface-01 border border-hairline p-s-5">…</div>

// Product surface
<div className="bg-l-surface-raised border border-l-border p-s-3">…</div>
```

The TS mirrors live in `tokens/`:

```ts
import { surfaceTokens, linearSurfaces } from "ui/tokens";

surfaceTokens.surface01;       // "var(--c-surface-01)"  — brand
linearSurfaces.surfaceRaised;  // "var(--l-surface-raised)" — product
```

## The one‑hot‑surface rule

`--c-ember` and `--l-p-high` resolve to the same hex (`#d8430a`) on
purpose. Whichever density you're in, **only one element on screen**
should glow ember at a time:

- The selected row in the timeline.
- The active item in a sidebar.
- The signal call‑to‑action ("Inject", "Run replay", "Block deploy").

Everything else stays neutral. If the whole UI glows, nothing glows.
